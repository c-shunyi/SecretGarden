# Secret Garden 基础技术文档

## 1. 文档目标
- 统一前后端技术基线，明确当前已落地能力与后续实现边界。
- 为 MVP（情侣记录生活 H5）提供可执行的开发规范。

## 2. 项目结构
- 前端项目：`secret-garden-h5`
- 后端项目：`secret-garden-server`
- 文档目录：`docs`

## 3. 技术选型

### 3.1 前端（已落地）
- `Vue 3`：页面开发框架
- `Vite`：本地开发与构建工具
- `Vue Router`：路由管理
- `ESLint + Oxlint`：代码检查

### 3.2 后端（已落地）
- `Node.js + Express`：HTTP 服务
- `cors`：跨域支持
- `dotenv`：环境变量管理
- `nodemon`：开发态热重启

### 3.3 数据库（技术基线）
- 数据库类型：`MySQL 8.x`（SQL 关系型数据库）
- ORM 方案：`Prisma`（建议）
- 迁移方式：`Prisma Migrate`（建议）


## 4. 整体架构
```text
H5 (Vue3 SPA)
   |
   | HTTPS / JSON
   v
Express API (Node.js)
   |
   | SQL
   v
MySQL
```

## 5. 前端工程规范

### 5.1 目录约定（当前）
- 入口：`secret-garden-h5/src/main.js`
- 路由：`secret-garden-h5/src/router/index.js`
- 页面：`secret-garden-h5/src/views`
- 全局样式：`secret-garden-h5/src/styles/base.css`

### 5.2 H5 适配
- `index.html` 中启用 viewport：`width=device-width, initial-scale=1.0, viewport-fit=cover`
- 使用 `base.css` 进行移动端基础重置（盒模型、字体、根节点高度等）
- 页面布局优先使用弹性布局（Flex）和响应式单位

### 5.3 代码规范
- 默认使用 `ES Module`
- 提交前执行：
  - `npm run lint`
  - `npm run build`

## 6. 后端工程规范

### 6.1 目录约定（当前）
- 入口：`secret-garden-server/src/server.js`
- 应用装配：`secret-garden-server/src/app.js`
- 路由：`secret-garden-server/src/routes`

### 6.2 API 规范
- 基础健康检查：
  - `GET /`
  - `GET /api/health`
- 返回格式统一为 JSON
- 业务 API 建议统一前缀：`/api/v1`

### 6.3 错误处理建议
- 统一错误响应结构：
```json
{
  "code": "ERROR_CODE",
  "message": "可读错误信息",
  "requestId": "可选，用于排查"
}
```

## 7. 数据库模型（MVP 建议）
- `users`：用户信息（账号、密码哈希、昵称、头像、生日、简介）
- `couples`：情侣关系
- `invitations`：邀请码
- `files`：文件中心表（上传元数据）
- `diaries`：日记主表
- `diary_files`：日记与文件关联表（1 条日记可关联多条文件记录）
- `diary_likes`：日记点赞
- `diary_comments`：日记评论
- `anniversaries`：纪念日

说明：
- 所有表建议包含：`id`、`created_at`、`updated_at`、`deleted_at(可选软删)`
- 文件模块详细设计见：`docs/文件功能技术方案.md`

## 8. 环境变量约定

### 8.1 当前
- 后端 `.env`：
  - `PORT=3000`

### 8.2 建议补充
- `NODE_ENV`
- `DB_HOST`
- `DB_PORT`
- `DB_USER`
- `DB_PASSWORD`
- `DB_NAME`
- `JWT_SECRET`
- `CORS_ORIGIN`

## 9. 开发与运行

### 9.1 前端
```bash
cd secret-garden-h5
npm install
npm run dev
```

### 9.2 后端
```bash
cd secret-garden-server
npm install
npm run dev
```
